# 목표
태스크 → AI 개발 → 결과 반영의 양방향 루프를 완성한다.

## 입력 방향 (태스크 → AI)
- [x] 태스크 본문에 PRD/스펙/설계 문서를 구조화하여 담을 수 있는 템플릿 또는 섹션 구조 — config.yml 템플릿 + CreateModal 템플릿 선택 UI
- [x] 태스크 내용이 Claude Code 훅을 통해 충분한 맥락으로 전달되는지 검증 — refs 필드가 plank-status.sh에서 주입됨
- [x] 연관 문서(프로젝트 룰, 코드 컨벤션 등)를 태스크에서 참조할 수 있는 구조 — refs 필드 + docs tab

## 출력 방향 (AI → 태스크)
- [x] AI 작업 완료 시 체크리스트 자동 업데이트 (현재 부분 구현됨) — CLAUDE.md에 승인 없이 자동 업데이트 규칙 명시
- [x] AI 작업 결과 요약이 태스크에 자동 기록되는 메커니즘 — CLAUDE.md에 체크리스트 항목에 결과 요약 덧붙이기 규칙 추가
- [x] 태스크 상태(컬럼) 자동 전환 제안 — CLAUDE.md에 모든 항목 완료 시 done 이동 제안 규칙 유지

## 검증
- [x] 실제 개발 루프에서 Plank만으로 태스크 작성 → AI 개발 → 결과 확인이 매끄러운지 테스트 — 현재 세션에서 실제로 이 루프를 사용 중

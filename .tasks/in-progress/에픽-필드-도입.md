## 목표
태스크 frontmatter에 `epic` 필드를 추가하여 라벨과 독립적으로 에픽(주제) 단위 그루핑 및 진행률 조감 구현.

## 배경
라벨은 태스크 유형(개발/디자인/버그)이고, 에픽은 프로젝트 주제(결제/정산/환불) 단위.
기존 라벨로 에픽 진행률을 보여주려 했으나 개념이 맞지 않아 분리 결정.

## 구현 항목
- [ ] 태스크 frontmatter에 `epic` 필드 추가 (예: `epic: payment`)
- [ ] config.yml에 epics 정의 (id, name, color)
- [ ] 태스크 생성/편집 모달에 에픽 선택 UI 추가
- [ ] 보드 상단에 에픽별 진행률 바 표시 (done 제외한 활성 태스크 기준)
- [ ] 에픽 필터와 라벨 필터 조합 지원

## 메모
- done 컬럼 누적 문제: 진행률 계산 시 활성 태스크만 카운트하거나, 기간 기반 필터 적용 검토
- 심층 인터뷰에서 도출된 요구사항

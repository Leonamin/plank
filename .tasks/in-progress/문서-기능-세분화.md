## 목표
현재 문서는 그냥 얼렁뚱땅 문서 경로 + 자유 입력 형식으로 되어있음

## 할일
- [ ] 글로벌 컨텍스트 문서, 이슈 추적 문서, 스키마, 플로우 등 기획에 필요한 문서가 무엇이 있을지 생각하고 그에 맞게 분류
- [ ] 문서 분류에 따라 배치를 다르게하고 눈에 보이게 해서 사용자로 하여금 다른 계층의 문서라는것을 보여주기
- [ ] 다음의 문서는 필수로 존재했으면 좋겠음
    - [ ] 글로벌 컨텍스트 문서: 현재 프로젝트가 무슨 프로젝트인지 도메인 설명
    - [ ] 기능 플로우 문서: 코드로만 보면 나중에 헷갈려서 플로우를 설명하는 기능 문서, 우린 Plank를 쓰니까 개발 후 자동 동기화도 편할거임
    - [ ] 이슈 트래킹 문서: 태스크를 통해 개발을 끝내도 좀 멀리 봐야하는 기능이 존재할 수 있음. 따라서 이슈 트래킹 문서는 기능 개발이 정말 다 끝날때까지, 또는 운영상 끝낼 수가 없어서 기록을 남겨놔야하는 내용을 적어두는 문서임. 태스크를 작성할 때, 알아서 해당 이슈 트래킹 문서를 임포트

## 방식
- 심층 인터뷰로 진행

# Plank — AI 연동 규칙

이 프로젝트는 Plank(마크다운 기반 칸반 보드)입니다. `.tasks/` 디렉토리에 태스크가 저장됩니다.

## Plank 원칙 준수

기능 추가, 설계 변경 시 반드시 `README.md`의 **Plank 원칙**을 확인한다. 원칙에 어긋나는 변경은 거부한다.

## Plank Integration

### 작업 시작 전
- `.tasks/` 내 태스크 파일들을 읽어 현재 진행 상황을 파악한다.
- `in-progress/`, `todo/` 컬럼의 태스크 중 사용자가 요청한 작업과 관련된 항목이 있는지 확인한다.
- 관련 태스크가 있으면 해당 태스크의 체크리스트와 의존성을 참고하여 작업한다.
- **관련 태스크가 없으면 구현을 시작하지 않는다.** 먼저 태스크 생성을 제안하고, 사용자 승인 후 태스크 파일을 만든 뒤 구현을 시작한다.
- 태스크 생성 시 구현할 항목을 체크리스트(`- [ ]`)로 미리 정리하여 작업 범위를 명확히 한다.

### 작업 완료 후
- 완료된 체크리스트 항목이 있으면 `- [ ]`를 `- [x]`로 업데이트한다.
- 태스크의 모든 항목이 완료되었으면 사용자에게 done 컬럼으로 이동할지 물어본다.
- 작업 중 발견한 새로운 TODO나 후속 작업이 있으면 사용자에게 태스크 생성을 제안한다.

### 태스크 파일 조작 규칙
- 태스크 파일은 `.tasks/{column}/{id}.md` 경로에 위치한다.
- 컬럼 이동은 파일을 해당 컬럼 디렉토리로 이동시킨다 (예: `todo/` → `in-progress/`).
- `done`으로 이동 시 주차 폴더를 생성한다 (예: `.tasks/done/2026-W12/`).
- 프론트매터 형식: `id`, `title`, `labels`, `priority`, `created`, `depends_on`.
- 새 태스크 ID는 제목을 slugify하여 생성한다.

### 논의 중 TODO 발생 시
- 사용자와 논의 중 구체적인 작업 항목이 나오면 "태스크로 등록할까요?"라고 제안한다.
- 승인 시 적절한 컬럼(기본 `backlog`)에 마크다운 파일을 생성한다.
